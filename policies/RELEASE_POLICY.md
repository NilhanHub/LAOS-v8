# Release policy

A release requires a clean source checkout, fully passing source-tree suite, deterministic clean staging, static/runtime separation, secret scan, SBOM, provenance, checksums, safe archive inspection, clean extraction, complete extracted-copy retest, clean installation test, real sandbox-provider validation, recorded real-agent evaluation, durable final storage, and final reopen/hash verification.

No completion statement may precede release evidence. Any P0/P1 defect, failed or missing required test, untested mandatory provider, missing artifact, or unverifiable claim blocks release.
