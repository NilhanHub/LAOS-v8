# Start Here for Nilhan

## What LAOS v8 is

LAOS v8 is your strong-AI-to-weaker-agent software delivery system. Give the **master LAOS system only to the highly capable Software Architect AI**. In practice, that will often be ChatGPT or another top-tier Architect model.

Do **not** give the master system, Architect Control Pack, full action graph, hidden checks, or signing key to a weaker desktop agent.

## New application

1. Give the strong Architect AI your approved goal, constraints, source materials, and repository location.
2. The Architect validates the specification and compiles a project pack.
3. Keep the `ARCHITECT_CONTROL_PACK` private with the Architect.
4. Give the weaker builder only the `AGENT_EXECUTION_PACK` and the current `CURRENT_ACTION` package or its single copy-pasteable prompt.
5. After the weaker agent returns its result, give the result and repository back to the Architect/runtime for verification.
6. Only after acceptance does the Architect issue the next action.

## Existing application

1. Give the strong Architect AI the repository and your desired outcome.
2. The Architect compiles a read-only Capture Execution Pack.
3. Give that capture pack to a weaker investigator.
4. Return the structured App Intelligence package to the Architect.
5. The Architect validates repository freshness, checks evidence, resolves conflicts, preserves unknowns, and forms its own understanding.
6. Only then does it compile continuation actions for weaker builders.

## Files you normally handle

- Private: `ARCHITECT_CONTROL_PACK/`
- Public to a weaker agent: `AGENT_EXECUTION_PACK/`
- Public for one action only: `CURRENT_ACTION/`
- Proof generated by execution: `<application-repository>/Evidence/`

## Safety rules

- Never share private signing keys.
- Never allow a weaker agent to see future actions or hidden tests.
- Never accept “done” without current checks, evidence, and required review.
- Require a real sandbox for high-risk work.
- Require explicit approval for deployments, payments, emails, publishing, destructive migrations, credential creation, or other consequential external actions.
- If LAOS reports `BLOCKED`, `NEEDS_ARCHITECT`, `NEEDS_RECAPTURE`, or `NEEDS_AMENDMENT`, do not override it with a broad prompt. Return control to the Architect.
